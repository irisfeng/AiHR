## 发现并解决的问题

- `prepare_job_offer_handoff` / `prepare_interviewer_pack` 在按钮调用时出现 `TimestampMismatchError`。
- 根因是 `validate` 钩子已自动回写候选人状态，API 又额外保存了一次候选人。
- 处理方式：
  - 保留 `validate` 钩子作为统一状态同步入口
  - 去掉 API 内重复的候选人保存
- 重灌 demo 数据时，原生 `Interview.Scheduled On` 因只写一次约束无法直接改动。
- 处理方式：
  - seed 逻辑改为“老记录不强改排期，只补必要字段”
  - 面试反馈提交失败时保留草稿，避免整批 demo 灌数回滚
- 命令栏和搜索栏仍残留旧工作台历史，如 `AIHR 招聘作战台`、`AIHR 面试官台`，且会生成已过时的入口。
- 根因是 `Route History` 仍保留旧路由记录，前端 `recent/frequent links` 会继续复用这些历史。
- 处理方式：
  - 新增 `aihr.setup.navigation`，把旧工作台历史统一改写到当前命名
  - 登录加载时通过 `extend_bootinfo` 自动清洗 `frequently_visited_links`
  - 前端 `aihr_desk.js` 增加命令栏历史净化，过滤所有 `Workspaces/*` 频繁访问项
  - 直接执行了一次数据库级 `cleanup_route_history`，确认 `Workspaces/HR` 已清空，旧 `AIHR 招聘作战台 / 用人经理台 / 面试官台` 已全部改写成新命名
- 右上角工作台切换和旧中文工作台 slug 仍会落到 `/app/aihr-招聘总览` 一类地址，并弹出“未找到页面”。
- 根因是 Frappe 前端会在部分入口里按中文工作台标题生成 slug，而直接打开这些旧中文 slug 时，服务端此前没有做兼容归一。
- 处理方式：
  - `aihr_desk.js` 为 `AIHR 招聘总览 / 用人经理中心 / 面试协同中心` 注册中文 slug 别名，并统一重写到稳定英文路由
  - 新增 `before_request` 级别的 desk 路径归一化，对 `/app` 和旧中文 slug 统一返回 `308` 到稳定工作台地址
  - 浏览器实测直接访问 `/app/aihr-招聘总览` 已会自动落到 `/app/aihr-hiring-hq`，不再弹“未找到页面”
- 登录后直接打开站点根路径或网站侧个人中心时，仍会出现 ERP/门户风格的“我的账户”页面，与 AIHR 主流程无关，容易让业务方误以为这是系统默认首页。
- 根因是 Frappe/ERPNext 自带网站门户的 `/me` 页面仍然可达，右上角用户菜单里原生也保留了 `查看网站`。
- 处理方式：
  - 对登录态的 `/me` 和门户别名统一重定向到 `AIHR 招聘总览`
  - 保留游客访问 `/` 时的登录页体验，但登录后网站侧会被前端脚本立即送回 Desk
  - Desk 右上角用户菜单移除 `查看网站`，并在前端兜底覆盖 `view_website` 动作，避免再次跳回门户页
- 左侧 sidebar 数据并未丢失，但默认 Frappe 工作台侧栏在 AIHR 视觉体系下显得过窄、过淡，远看会像空白区。
- 浏览器实测当前侧栏实际已加载 3 个工作台入口：
  - `AIHR 招聘总览`
  - `AIHR 面试协同中心`
  - `AIHR 用人经理中心`
- 处理方式：
  - 加强侧栏容器宽度、分组标题对比度、项目高度与文字可读性
  - 将 `Public` 分组名明确显示为 `AIHR 导航`
  - 隐藏工作台编辑拖拽控制，减少左侧噪音，让它更像正式导航栏
- 左侧顶部仍有一条看起来像“空白占位”的白框。
- 根因是 Frappe 工作台在只有一个可见分组时，仍然保留了分组标题按钮；在 AIHR 侧栏里这块既没有信息量，也会被误认为是空白输入框。
- 处理方式：
  - 当侧栏只剩一个可见分组时，自动隐藏该分组标题按钮
  - 左侧导航直接从第一个工作台入口开始显示，减少误解
