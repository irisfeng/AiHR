# AIHR HR 专员 MVP 使用手册

## 1. 这套系统解决什么问题

这版 AIHR 不是完整 HRMS，也不是旧 Frappe 站点的换皮。

它只解决一条招聘执行主线，目标是给 `HR 专员` 减负提效：

1. 从用人经理拿到自由文本需求
2. 在系统里整理成岗位需求单并生成 JD
3. 把 JD 发给代理商并记录外发动作
4. 上传代理商返回的 `ZIP` 简历包，由 AI 自动初筛
5. 发给用人经理复核
6. 约面试、追反馈、做录用收口
7. 导出候选人总表，查看代理商质量

## 2. 角色边界

### HR 专员

主操作者，负责：

- 录入经理原话
- 补齐岗位需求
- 生成 JD
- 记录代理外发
- 上传简历包
- 跟进经理复核
- 安排面试
- 追反馈
- 录用收口
- 导出总表和看代理商质量

### 用人经理

只做判断，不进完整后台。经理页只允许：

- `同意推进`
- `要求补充信息`
- `暂不推进`

## 3. 本地启动

安装依赖：

```bash
pnpm install
pnpm install:api
```

启动 API：

```bash
pnpm dev:api
```

启动 Web：

```bash
pnpm dev:web
```

默认地址：

- Web: `http://localhost:3000`
- API: `http://127.0.0.1:8000`

这条直重构主线不需要 Docker。

## 4. 页面说明

### `/`

`待办`

HR 默认首页。按动作分组展示待办：

- 待整理需求
- 待生成 / 确认 JD
- 待发代理
- 待处理简历包
- 待经理复核
- 待约面试 / 待补录用资料

### `/jobs`

`岗位需求`

用于：

- 粘贴经理原始需求
- 生成岗位需求单初稿
- 生成 JD
- 记录代理外发

### `/candidates`

`候选人`

用于：

- 上传 `ZIP` 简历包
- 查看最近导入候选人
- 跟踪待经理复核
- 下载候选人总表

### `/manager/reviews`

`用人经理复核入口`

给经理使用的极简页面，只做推进/补充/暂不推进判断。

### `/interviews`

`面试与录用`

用于：

- 安排面试
- 跟进待反馈面试
- 回填反馈
- 做 Offer 与录用收口

### `/settings`

`设置与模板`

用于：

- 查看代理商质量评估
- 统一岗位和交付口径

## 5. 标准操作流程

## 流程 A：从经理原话生成 JD

1. 打开 `/jobs`
2. 在 `岗位需求采集与 JD 生成` 中粘贴经理原话
3. 点击 `生成岗位需求单初稿`
4. 检查右侧抽取字段和 `缺失信息`
5. 点击 `生成 JD`

结果：

- 状态从 `待整理 / 待确认 JD` 进入 `待发代理`
- 系统会同步创建或更新关联岗位

## 流程 B：记录代理外发

1. 在 `/jobs` 的 `代理外发记录` 里选择岗位
2. 填写代理商、发送时间、备注
3. 点击 `记录已发代理`

结果：

- 岗位状态进入 `代理寻访中`
- 外发动作留痕，不再依赖 Excel

## 流程 C：上传 ZIP 简历包

1. 打开 `/candidates`
2. 在 `ZIP 简历导入与 AI 初筛` 中选择目标岗位
3. 上传代理商给的 `ZIP`
4. 点击 `上传并后台解析`

系统会自动：

- 解压 ZIP
- 解析 PDF / DOCX / DOC / TXT
- 创建候选人
- 生成初筛结果
- 更新岗位漏斗

## 流程 D：发给经理复核

1. HR 在 `/candidates` 查看 `待经理复核`
2. 经理打开 `/manager/reviews`
3. 经理在 `复核动作` 中做选择：
   - 推进
   - 补充信息
   - 暂不推进

如果选择 `推进` 且保留默认勾选：

- 系统会直接创建一条面试记录
- 候选人状态会进入 `面试中`

## 流程 E：安排面试与追反馈

1. 打开 `/interviews`
2. 在 `快速安排面试` 中填写轮次、时间、面试官
3. 在 `待追回反馈` 中看哪些面试超时未回
4. 在 `回填面试反馈` 中提交结论

结果：

- 候选人下一步动作会同步更新
- 候选人时间线会自动写入反馈记录

## 流程 F：录用收口

1. 在 `/interviews` 的 `录用收口单` 中创建 Offer 交接
2. 填写薪资期望、薪酬备注、入职负责人、薪酬负责人
3. 后续点击 `标记薪酬就绪`

用于统一记录：

- 薪酬待遇
- 入职推进
- 资料收口状态

## 流程 G：导出候选人总表

1. 打开 `/candidates`
2. 在 `候选人总表导出` 中按关键字、代理商、最终去向筛选预览
3. 点击 `下载候选人总表`

说明：

- 总表包含所有进入系统的候选人
- 无论录用、未录用、暂缓，都保留记录

## 流程 H：查看代理商质量

1. 打开 `/settings`
2. 查看 `代理商质量评估`

当前指标：

- 推荐简历数
- 初筛通过率
- 经理通过率
- 面试转化率
- Offer 转化率
- 最终录用率

## 7. 常见问题

### 页面显示 `Fallback Data`

说明 Web 没连上 FastAPI。

检查：

```bash
curl http://127.0.0.1:8000/healthz
```

### 上传 ZIP 没反应

确认：

- API 已启动
- 上传的是 `.zip`
- 压缩包里是支持的简历格式

### 导出按钮不可用

候选人总表下载依赖 Live API。若当前页面是 fallback 模式，需要先启动 API。

## 8. 当前已知边界

- 目前默认是本地单站点试用，不含正式认证
- 批量导入主路径是 `ZIP`
- 代理商评估是轻量事实指标，不是复杂 BI
- 仍未接入正式 Postgres、对象存储和权限体系
