# AIHR 国际化状态说明

## 当前结论

AIHR **还没有完成完整的中英文国际化**，当前状态更准确地说是：

**“中文优先体验已开始落地，但还不是完整的中英双语系统。”**

这意味着：

- 面向国内团队演示和试用，当前已经基本够用
- 如果要做到“用户可自由切换中英文、全站术语统一、字段标签双语一致”，还需要继续补

## 已经完成的部分

### 1. 中文优先的操作体验

当前 MVP 中，用户最常看到的关键操作已经改成中文：

- 候选人页按钮
  - `运行 AI 初筛`
- 岗位需求页按钮
  - `刷新代理发布包`
  - `复制代理发布包`
- 岗位开启页按钮
  - `同步代理发布包`
  - `批量 AI 初筛`
  - `查看候选人概览`

### 2. 中文优先的 AI 输出

AI 初筛相关的核心输出已经改成中文优先：

- AI 摘要
- 优势
- 风险点
- 建议追问问题
- 代理发布包文案

### 3. 中文简历解析已开始增强

当前已支持部分中文简历解析：

- 中文姓名识别
- 中文城市识别
- 中文工作年限识别
- 中文技能词识别

### 4. Frappe / HRMS 自带中文基础能力可复用

底座框架本身支持多语言，当前站点在中文环境下已经能显示不少中文界面元素，例如登录页、基础菜单和部分标准模块文案。

## 还没有完成的部分

### 1. 还没有建立 `aihr` 自己的翻译文件

当前仓库里还没有以下标准国际化资源：

- `aihr/translations/zh.csv`
- `aihr/translations/en.csv`
- `.po` / `.mo`

这意味着现在很多中英文切换还不是通过正式翻译机制完成的，而是直接写在按钮或 AI 文案里。

### 2. 自定义字段标签仍然大量是英文

例如这些字段标签目前仍然主要是英文：

- `Salary Currency`
- `Agency Brief`
- `Match Score`
- `Candidate City`
- `Next Action`

也就是说：

- 页面主操作已经偏中文
- 但字段元数据层还没有全面中文化

### 3. 底层状态源值仍然是英文

例如当前仍然保留英文源值：

- `Advance`
- `Ready for Review`
- `Hold`
- `Screened`

前端展示层已经开始把它们翻译为中文标签，但数据库和代码判断仍使用英文源值。

这是当前阶段的正确做法，因为它更利于：

- 后续做中英切换
- API 集成
- Agent 调用
- 升级和排错

### 4. 还没有完成“用户级语言切换”

当前还没有专门为 AIHR 做这些内容：

- 中文默认 / 英文可切换的系统设置说明
- 按用户语言显示不同 AI 输出
- 按语言切换字段、菜单、帮助文案

## 推荐目标状态

AIHR 后续应做成：

**“中文默认，英文可切换，底层技术字段保持英文。”**

推荐分层如下：

### 技术层保留英文

- DocType 名称
- `fieldname`
- API 参数
- 数据库存储状态值

### 显示层支持双语

- 页面标题
- 字段标签
- Workspace
- 按钮
- 提示语
- AI 输出
- 通知模板

## 对当前项目的判断

如果以“国内 HR 团队先试用”为目标，当前中文程度已经足够支持 MVP 演示和试跑。

如果以“正式中英双语产品”作为标准，当前还**没有完成**。

## 下一步建议

推荐按下面顺序推进国际化，而不是现在就大面积重构：

1. 先保持当前 MVP 主链路继续开发
2. 再补 `aihr/translations/` 正式翻译资源
3. 把自定义字段标签和 Workspace 文案统一接入翻译
4. 给 AI 输出增加 `lang/locale` 参数
5. 最后再做英文模式验证

## 一句话总结

当前 AIHR 的国际化状态是：

**不是完整的中英文国际化，但已经完成了中文优先的 MVP 体验层。**
