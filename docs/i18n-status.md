# AIHR 国际化状态说明

## 当前结论

AIHR **还没有完成完整的中英文国际化**，当前状态更准确地说是：

**“中文优先体验已开始落地，但还不是完整的中英双语系统。”**

这意味着：

- 面向国内团队演示和试用，当前已经基本够用
- 如果要做到“用户可自由切换中英文、全站术语统一、字段标签双语一致”，还需要继续补

## 已经完成的部分

### 1. 中文优先的操作体验

当前 MVP 中，用户最常看到的关键操作已经改成中文：

- 候选人页按钮
  - `运行 AI 初筛`
- 岗位需求页按钮
  - `刷新代理发布包`
  - `复制代理发布包`
- 岗位开启页按钮
  - `同步代理发布包`
  - `批量 AI 初筛`
  - `查看候选人概览`
- 标准列表页和表单页骨架
  - `岗位需求单`
  - `新建岗位需求单`
  - `AI 初筛`
  - `新增 AI 初筛`
  - `列表视图`
  - `过滤条件`
  - `尚未保存`
  - `保存`
  - `编号规则`
  - `预期薪酬`
  - `需求提出人`
  - `时间安排`
  - `发布日期`

### 2. 中文优先的 AI 输出

AI 初筛相关的核心输出已经改成中文优先：

- AI 摘要
- 优势
- 风险点
- 建议追问问题
- 代理发布包文案

### 3. 中文简历解析已开始增强

当前已支持部分中文简历解析：

- 中文姓名识别
- 中文城市识别
- 中文工作年限识别
- 中文技能词识别

### 4. Frappe / HRMS 自带中文基础能力可复用

底座框架本身支持多语言，当前站点在中文环境下已经能显示不少中文界面元素，例如登录页、基础菜单和部分标准模块文案。

### 5. AIHR 增加了前端可见层兜底翻译

除了正式的 `zh.csv` 词条之外，AIHR 还在 Desk 层补了一层前端可见文案兜底：

- 页面标题
- 面包屑
- 列表页动作按钮
- 筛选器标题
- `title` / `aria-label` / `placeholder`

这样可以覆盖一部分上游组件没有稳定走翻译链的场景。

## 还没有完成的部分

### 1. `aihr` 自己的翻译文件已经开始建立

当前仓库里已经补上第一版正式翻译资源：

- `aihr/translations/zh.csv`

这意味着：

- 自定义字段标签
- 自定义状态词
- `AI Screening` DocType 标签

这些内容已经可以开始走 Frappe 的正式翻译机制，而不只是依赖硬编码中文。

但也还没有完全做完：

- 还没有 `en.csv`
- 还没有 `.po` / `.mo`
- 还没有覆盖全部客户端字符串和帮助文案

### 2. 自定义字段标签仍然需要继续覆盖

例如这些字段标签目前仍然主要是英文：

- `Salary Currency`
- `Agency Brief`
- `Match Score`
- `Candidate City`
- `Next Action`

也就是说：

- 页面主操作已经偏中文
- 字段元数据层已经开始通过 `zh.csv` 收口
- 但还没有做到全量覆盖

例如今天虽然已经把招聘主链路里最常见的页面收到了中文，但仍可能看到零散残留：

- 富文本编辑器工具条
- 个别 tooltip
- 少数系统级辅助提示

### 3. 底层状态源值仍然是英文

例如当前仍然保留英文源值：

- `Advance`
- `Ready for Review`
- `Hold`
- `Screened`

前端展示层已经开始把它们翻译为中文标签，但数据库和代码判断仍使用英文源值。

这是当前阶段的正确做法，因为它更利于：

- 后续做中英切换
- API 集成
- Agent 调用
- 升级和排错

### 4. 还没有完成“用户级语言切换”

当前还没有专门为 AIHR 做这些内容：

- 中文默认 / 英文可切换的系统设置说明
- 按用户语言显示不同 AI 输出
- 按语言切换字段、菜单、帮助文案

### 5. 默认语言已开始向中文收口

当前安装逻辑已开始将以下默认值向中文收口：

- `System Settings.language`
- `User.language`（空值或英文默认值）

这能保证国内团队第一次打开系统时，不会因为默认语言还是英文而看到大量英文元数据。

## 推荐目标状态

AIHR 后续应做成：

**“中文默认，英文可切换，底层技术字段保持英文。”**

推荐分层如下：

### 技术层保留英文

- DocType 名称
- `fieldname`
- API 参数
- 数据库存储状态值

### 显示层支持双语

- 页面标题
- 字段标签
- Workspace
- 按钮
- 提示语
- AI 输出
- 通知模板

## 对当前项目的判断

如果以“国内 HR 团队先试用”为目标，当前中文程度已经足够支持 MVP 演示和试跑。

如果以“正式中英双语产品”作为标准，当前还**没有完成**。

## 下一步建议

推荐按下面顺序推进国际化，而不是现在就大面积重构：

1. 先保持当前 MVP 主链路继续开发
2. 继续扩充 `aihr/translations/zh.csv`
3. 把客户端 JS 里的中文字符串改成可翻译写法
4. 给 AI 输出增加 `lang/locale` 参数
5. 再补英文模式与用户级切换验证

## 一句话总结

当前 AIHR 的国际化状态是：

**不是完整的中英文国际化，但已经进入“中文翻译资源 + 中文默认语言 + 前端可见层兜底”阶段。**
