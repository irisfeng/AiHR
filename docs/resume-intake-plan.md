# 简历导入与优化方案

## 目标

AIHR 的简历入口应当围绕真实招聘动作设计：

- 简历多由供应商线下或其他渠道打包提供
- HR 常收到的是一个 `ZIP` 压缩包，而不是逐份上传
- 系统需要自动解析、自动建档、自动生成摘要，而不是把录入工作继续丢给 HR

因此，这条能力的目标应当定义为：

**供应商简历包上传 -> 自动解析 -> 候选人建档 -> AI 摘要 -> 进入招聘主链路**

## 产品原则

### 不做公司内部排行榜

`能量点 / 审核点 / 排名 / 月度排名 / 排行榜` 来自上游 Frappe 自带的用户画像逻辑，不属于 AIHR 当前产品范围。

对 AIHR 来说：

- 不建议做 HR、经理、面试官个人排名
- 不建议把内部协作做成“卷指标”
- 不建议把用户活跃度误当成人力价值

当前更应该关注的是：

- 岗位推进效率
- 候选人质量
- 面试反馈时效
- 供应商交付质量

### 供应商可以评估，但不做 PK 式榜单

如果后续需要评估外部招聘供应商，建议做质量看板，不建议做刺激性的排行榜。

推荐维度：

- 简历有效率
- 简历去重率
- 面试转化率
- Offer 转化率
- 到岗转化率
- 首批简历交付时效
- JD 匹配质量

推荐名称：

- `供应商概览`
- `供应商交付质量`
- `供应商服务表现`

不建议名称：

- `排行榜`
- `PK 榜`
- `本月第一`

## 当前实现评估

### 之前的状态

在本轮优化前，系统更接近“单份候选人录入”：

- 支持单份简历文本
- 支持单个附件解析
- 不支持供应商 `ZIP` 批量导入
- 对 `DOCX` 的支持不完整
- AI 摘要主要依赖当前本地初筛逻辑

所以，严格来说，之前**不符合**你提出的真实业务场景。

### 当前已经落地

现在这条链路已经补到 MVP 可用状态，支持：

- 在 `招聘中岗位` 页面点击 `导入简历压缩包`
- 上传供应商提供的 `ZIP` 文件
- 批量解析压缩包中的：
  - `PDF`
  - `DOCX`
  - `DOC`
  - `TXT`
- 自动提取正文
- 自动创建或更新 `Job Applicant`
- 自动挂接到当前 `Job Opening`
- 记录：
  - 原始文件名
  - 供应商名称
  - 来源渠道
  - 导入批次
  - 解析状态
- 可选自动触发 `AI 初筛`

### 当前仍有边界

当前的 `AI 初筛` 已经能输出：

- 摘要
- 风险点
- 匹配技能
- 建议追问问题
- 匹配分

但它目前仍以系统内第一版摘要逻辑为主，**还不是完整的大模型在线摘要服务**。

所以最准确的说法是：

**当前已支持 ZIP 批量导入与自动摘要第一版，但还未完成标准化大模型摘要引擎。**

## 当前推荐操作步骤

1. 打开目标 `招聘中岗位`
2. 点击 `导入简历压缩包`
3. 上传供应商提供的 ZIP 文件
4. 填写 `供应商名称`
5. 填写 `来源渠道`
6. 选择是否自动生成 `AI 摘要`
7. 导入后进入候选人列表查看入库结果
8. 打开候选人详情页查看 AI 摘要卡
9. 继续推进面试、反馈、Offer 和入职交接

## 当前最合适的文件约定

建议供应商提供的压缩包遵循这些约定：

- 一个 ZIP 对应一个岗位
- 文件名尽量包含候选人姓名
- 优先提供文本型 `PDF` 或 `DOCX`
- 尽量不要只给图片扫描件
- 不要把多个岗位的简历混在同一个 ZIP 里

## 下一步优化方案

### P0 已完成

- 收口 AIHR 中与内部排行榜相关的干扰入口
- 补齐 `ZIP` 批量简历导入
- 增强 `PDF / DOCX / DOC / TXT` 文本提取
- 支持候选人自动建档与导入批次标记
- 支持导入后自动触发 `AI 初筛`

### P1 建议继续做

- 增加导入结果页
- 展示成功数、失败数、跳过数、不支持格式数
- 增强去重规则
- 增加导入日志下载
- 增加供应商维度的导入统计

### P2 升级为标准大模型摘要

建议下一阶段统一做成标准摘要管线：

1. 文件正文提取
2. 结构化字段抽取
3. 调用大模型生成候选人摘要
4. 输出匹配点、风险点、建议追问
5. 回写到候选人摘要卡和经理视图

建议统一输出：

- 候选人概述
- 工作年限
- 行业背景
- 目标岗位匹配度
- 核心技能命中
- 风险提示
- 薪资与地点信息
- 建议面试关注点

## 结论

AIHR 当前应坚持两条原则：

- 内部不做排行榜文化
- 简历入口以供应商 ZIP 批量导入为主

当前 MVP 已经向这个方向完成了第一版落地，并可继续沿着：

**ZIP 批量导入 -> 自动解析 -> 候选人建档 -> AI 摘要 -> 招聘推进**

这条路径持续增强。
