# AIHR MVP 测试操作手册

## 文档目的

这份文档用于当前 AIHR MVP 的本地测试、内部演示和验收。

文档目标不是介绍概念，而是提供一份可以直接照着执行的测试步骤。

适用对象：

- HR
- 用人经理
- 面试官
- 实施与验收人员

```bash

```



## 二、快速验收

这部分用于 15 到 20 分钟内快速确认系统当前可用。

## 步骤 1：验证登录页

操作：

1. 打开 `http://development.localhost:18000/login`
2. 查看登录页是否为左右双栏布局
3. 左侧应为登录区域
4. 右侧应为 AIHR 说明卡片

预期结果：

- 页面不是 ERP 默认登录页
- 页面标题应显示 `AIHR 登录`
- 布局不是上下堆叠
- 背景为浅色，不是黑底

## 步骤 2：验证登录页的其他入口

操作：

1. 在登录页点击 `通过邮件链接登录`
2. 确认页面仍为左右双栏布局
3. 点击 `返回登录`
4. 点击 `忘记密码`
5. 再次确认页面仍为左右双栏布局

预期结果：

- `通过邮件链接登录`
- `忘记密码`

这两个页面都应该保持同一套 AIHR 登录壳，不应变成上下错位。

说明：

- 当前本地测试环境不要求真实邮件收发成功
- 本步骤主要验证页面结构、入口和跳转正确

## 步骤 3：登录系统

操作：

1. 返回登录页
2. 输入用户名：`Administrator`
3. 输入密码：`AIHRAdmin!2026`
4. 点击 `登录`

预期结果：

- 登录后进入 `AIHR 招聘总览`
- 不应进入 ERP 会计、项目或个人门户页面
- 地址应落到 `/app/aihr-hiring-hq`

## 步骤 4：验证首页与左侧导航

操作：

1. 观察左侧导航
2. 观察首页主卡和指标区
3. 检查是否可见以下入口：
   - `AIHR 招聘总览`
   - `AIHR 用人经理中心`
   - `AIHR 面试协同中心`

预期结果：

- 左侧导航只保留 AIHR 相关入口
- 不应出现会计、库存、排行榜、用户画像之类无关模块
- 首页主标题应为 `招聘总览`

## 步骤 5：验证角色工作台切换

操作：

1. 点击左侧 `AIHR 用人经理中心`
2. 查看页面是否正常打开
3. 再点击 `AIHR 面试协同中心`
4. 再切回 `AIHR 招聘总览`

预期结果：

- 三个工作台都能正常切换
- 不应出现“未找到页面”
- 不应跳到中文错误 slug 页面

## 测试前准备

请自行准备一个 ZIP 文件，压缩包中建议放入 2 到 5 份简历。

建议格式：

- `PDF`
- `DOCX`
- `DOC`
- `TXT`

建议：

- 使用新的候选人姓名和邮箱，避免覆盖当前演示数据
- 最好至少准备：
  - 1 份 PDF
  - 1 份 DOCX
  - 1 份中文简历

当前仓库中没有内置真实简历样例包，因此这一步需要测试人员自备 ZIP 文件。

## ZIP 导入步骤

操作：

1. 打开 `HR-OPN-2026-0001`
2. 点击 `导入简历压缩包`
3. 选择本地 ZIP 文件
4. 输入供应商名称
5. 输入来源渠道
6. 选择是否自动生成 `AI 摘要`
7. 提交导入

预期结果：

- 系统能接收 ZIP 文件
- 系统会逐份解压并解析内部简历
- 系统会为每份简历创建或更新 `Job Applicant`
- 每个候选人会保留来源信息
- 如果勾选自动生成 AI 摘要，导入后应出现对应 `AI 初筛` 结果

## 导入后验证

操作：

1. 进入 `Job Applicant` 列表
2. 查看是否新增候选人
3. 打开新增候选人
4. 检查以下信息：
   - 姓名
   - 邮箱
   - 手机号
   - 城市
   - 工作年限
   - 简历文本
   - AI 摘要
   - 供应商
   - 来源渠道

预期结果：

- 至少能生成候选人基础档案
- 解析成功的简历应能提取出结构化信息
- AI 摘要应能帮助经理快速理解候选人，而不必先看原简历

说明：

- `DOC` 当前属于 best-effort 解析，不同文件质量可能影响结果
- 若 ZIP 中存在重名或相同邮箱的候选人，系统可能走“更新已有候选人”逻辑

## 四、完整业务回归

如果需要从头完整跑一遍招聘主链路，建议按下面顺序操作：

1. 新建一个新的 `岗位需求单`
2. 创建对应的 `招聘中岗位`
3. 通过 ZIP 导入或手工录入候选人
4. 生成 `AI 初筛`
5. 选择一个候选人安排面试
6. 填写 `面试反馈`
7. 生成 `Offer`
8. 完成薪酬交接
9. 进入 `入职交接`
10. 创建 `Employee` 主档

如果只是做日常验收，优先使用本手册前面的演示数据即可，不必每次都从头新建一套数据。

## 五、建议验收标准

通过标准建议至少满足以下 8 条：

1. 登录页和邮件链接登录页布局正常
2. 登录后默认进入 `AIHR 招聘总览`
3. 左侧导航只显示 AIHR 入口
4. 岗位需求、招聘中岗位、候选人、AI 初筛可以正常打开
5. 面试、面试反馈、Offer、入职交接可以正常打开
6. `Offer -> 入职交接 -> Employee` 主档链路可追踪
7. ZIP 简历包可上传并触发候选人建档
8. 关键界面中文显示正常，无明显 ERP 干扰页面

## 六、常见问题

### 1. 页面样式看起来不对

先执行：

- 关闭当前标签页
- 重新打开页面
- 强制刷新：`Cmd + Shift + R`

### 2. 点击后跳到旧页面或错误页面

请从以下入口重新进入：

- `http://development.localhost:18000/login`
- `http://development.localhost:18000/app/aihr-hiring-hq`

### 3. 通过邮件链接登录是否要求本地真的收到邮件

当前本地测试不要求真实 SMTP 收发成功。

测试重点是：

- 页面结构正确
- 跳转正确
- 文案与交互正常

### 4. ZIP 导入后没有新增候选人

优先检查：

- ZIP 内部文件格式是否受支持
- 文件内容是否可解析
- 候选人是否因重复邮箱被更新而不是新建
